本项目通过 [Create React App](https://github.com/facebook/create-react-app)启动。

## 可用脚本

在项目目录下，你可以运行

### `npm start`

以开发模式运行应用程序。
打开 [http://localhost:3000](http://localhost:3000)，在浏览器中查看。

当你进行更改时，页面会重新加载。
你还可以在控制台中看到任何 lint 错误。

###`npm test``

以交互式观察模式启动测试运行器。
更多信息请参阅[运行测试](https://facebook.github.io/create-react-app/docs/running-tests)部分。

### `npm run build`.

在 `build` 文件夹中构建用于生产的应用程序。
它会在生产模式下正确捆绑 React，并优化构建以获得最佳性能。

编译后的文件将被最小化，文件名包含哈希值。
您的应用程序已准备就绪，可以部署了！

更多信息，请参阅 [部署](https://facebook.github.io/create-react-app/docs/deployment) 部分。

### `npm run eject`.

**注意：这是一个单向操作。一旦 “弹出”，就无法返回！ **

如果你对构建工具和配置选择不满意，可以随时`eject`。该命令将删除项目中的单一构建依赖关系。

相反，它会将所有配置文件和传递依赖（webpack、Babel、ESLint 等）直接复制到你的项目中，因此你可以完全控制它们。除了 “eject ”之外，其他所有命令都将继续工作，但它们将指向复制的脚本，因此你可以对它们进行调整。在这一点上，你就可以独立操作了。

你不必再使用 `eject`。精心策划的功能集适用于中小型部署，你不应该觉得必须使用这一功能。不过我们知道，如果您不能在准备就绪时对其进行自定义，那么该工具就不会有用。
